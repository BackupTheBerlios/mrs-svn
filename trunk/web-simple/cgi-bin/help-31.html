Creating your own datafiles

You can create MRS datafiles by first downloading the mrs.pl driver script. This script uses parser plugins to do the actual parsing and handles all bookkeeping. It assumes you have set up your data directories in the same way as we did at the CMBI. This means you have some base directory containing the directories 'raw' and 'mrs'. In the raw directory there are directories for each databank to be indexed.


.../data/
     +---  mrs/
     +---  raw/
            +---   embl/
            +---   uniprot/
            +---   pdb/
            ...

The next thing you need to do is set three environmental variables. MRS_SCRIPT_DIR should point to the directory containing your parser scripts. MRS_RAW_DIR should point to the .../data/raw/ directory and MRS_DATA_DIR should point to the .../data/mrs/ directory.

And now you can use mrs.pl to create datafiles. Suppose you want to update your uniprot, all you have to type is:

./mrs.pl create -d uniprot

This command will start uncompressing and parsing the raw uniprot files in .../data/raw/uniprot/ and will create a lot of temporary files called uniprot.cmpXXxxxxxetc in .../data/mrs/. Once parsing and indexing is done the temp files will be deleted and the old uniprot.cmp file will be replaced with the new one.

mrs.pl has a number of options. You can see all of them by simply typing the mrs.pl command without parameters.

A set of parsers is supplied with the mrsweb tarball and can be found in /usr/local/MrsWeb/parsers/ assuming you installed the MRS website in /usr/local.
