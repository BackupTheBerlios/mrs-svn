Installing MRS locally

Requirements

You need Perl version 5.8 compiled for multi threading.

Obtaining the software

To install the MRS website locally you need to first download the mrsweb.tgz file containing the website scripts and parser scripts. You then need to download the MRS Perl plugin. There are currently three versions of this plugin, one for Linux-i686 built for Linux and Intel Pentium 4 or compatible chips, one for MacOS X 10.3 and one for Linux x86_64, a 64 bit Linux version.

Installing the plugin

The plugin can be installed by extracting the plugin archive and then run the usual Perl make commands like this:


tar xvzf MRS-linux-x86_64-1.0.0.tgz
cd MRS-1.0.0
perl Makefile.PL
make
make install

You will probably have to run these commands as root

Setting up the website

Simply extract the mrsweb.tgz file at the location where you want to put it. Then edit your local apache config file to point to this directory. Suppose you installed the mrsweb at /usr/local/MrsWeb you would have to add the following lines to httpd.conf:


ScriptAlias /mrs/cgi-bin/ /usr/local/MrsWeb/cgi-bin/
Alias /mrs/ /usr/local/MrsWeb/htdocs/

After editing this file you should run apachectl restart and then the MRS website should be available at http://localhost/mrs/

Configuring MRS

The last thing you need to do to make the website work is define where to put the MRS datafiles and tell MRS about it. You do so by creating an mrs.conf file in the cgi-bin directory of MRS. A template is suplied with the package, copy the mrs.conf.default file to mrs.conf and open this file with your favorite editor. Replace the word __DATA_DIR__ with the path to where you want to store the MRS data files.

Another thing you need to do is edit this mrs.conf file to reflect the files you have locally. Simply remove references to the datafiles you don't have.
